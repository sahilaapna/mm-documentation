API Documentation

Base URL​

https://qaintegrationapi.meramonitor.com

Authentication – Get Token​

Generates an access token using client credentials.

Endpoint

POST /api/oauth/token

Content-Type

application/x-www-form-urlencoded

Request Parameters

Parameter Type Required Description
client_id string Yes Your client ID.
client_secret string Yes Your client secret.
grant_type string Yes Must be set to client_credentials.
scope string Yes Must be set to api.

Sample Request


curl --location --request POST 'https://qaintegrationapi.meramonitor.com/api/oauth/token' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'client_id=YOUR_CLIENT_ID' \
--data-urlencode 'client_secret=YOUR_CLIENT_SECRET' \
--data-urlencode 'grant_type=client_credentials' \
--data-urlencode 'scope=api'

Sample Response


{
    "token_type": "Bearer",
    "expires_in": 31536000,
    "access_token": "YOUR_ACCESS_TOKEN"
}

Check-In/Check-Out - All Users​

Retrieve check-in/check-out records of all users in the organization.

Endpoint

GET /api/attendance/checkincheckout

Authentication

Send the token in the Authorization header as a Bearer token.

Authorization: Bearer {access_token}

Query Parameters

Parameter Type Required Description Example
date string Yes Date for which records are to be fetched (YYYY-MM-DD). 2024-07-30
page integer No Page number for pagination. Default is 1. 1

Sample Request


curl --location --request GET 'https://qaintegrationapi.meramonitor.com/api/attendance/checkincheckout?date=2024-07-30&page=1' \
--header 'Authorization: Bearer YOUR_ACCESS_TOKEN'

            "total_hours": "8:30"

Check-In/Check-Out - Specific User​

Retrieve check-in/check-out records for a specific user.

Endpoint

GET /api/attendance/checkincheckout/user

Authentication

Send the token in the Authorization header as a Bearer token.

Authorization: Bearer {access_token}

Query Parameters

Parameter Type Required Description Example
date_from string Yes Start date for the records (YYYY-MM-DD). 2024-07-01
date_to string Yes End date for the records (YYYY-MM-DD). 2024-07-30
user_id integer Yes ID of the user. 123
page integer No Page number for pagination. Default is 1. 1

Sample Request


curl --location --request GET 'https://qaintegrationapi.meramonitor.com/api/attendance/checkincheckout/user?date_from=2024-07-01&date_to=2024-07-30&user_id=123&page=1' \
--header 'Authorization: Bearer YOUR_ACCESS_TOKEN'

{

            "name": "Jane Smith",

            "date": "2024-07-30",
            "checkin_time": "09:15:00",
            "checkout_time": "17:45:00",
            "total_hours": "8:30"
